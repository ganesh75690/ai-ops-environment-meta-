# Task Scenarios

This document outlines example scenarios used in the environment to evaluate agent behavior.

---

## 🟢 Scenario 1 — Easy Task

### Input
- Task: Resolve low-priority customer query
- Context: Standard issue, no escalation needed

### Expected Behavior
- Agent assigns task correctly
- No escalation
- Quick resolution

### Success Criteria
- Correct classification
- Efficient handling

---

## 🟡 Scenario 2 — Medium Task

### Input
- Task: Handle delayed service issue
- Context: Customer dissatisfaction increasing

### Expected Behavior
- Agent prioritizes task appropriately
- May escalate if needed
- Takes structured decision

### Success Criteria
- Proper prioritization
- Logical decision-making

---

## 🔴 Scenario 3 — Hard Task

### Input
- Task: Critical system failure
- Context: High urgency, multiple dependencies

### Expected Behavior
- Immediate escalation
- Correct resource allocation
- Strategic decision-making

### Success Criteria
- Fast response
- Correct escalation
- Optimal handling of complexity

---

## Summary

These scenarios simulate real-world operational decision-making with increasing complexity, enabling evaluation of agent performance across difficulty levels.
